Data bases (v 0.1)
------------------

We will define the main data base and tables like next:

Table TagLinker
- Version       The current version
- 

Table Element:
- ID primary key (unique / auto increment).
- Type (enum to define the type: Contact | tag | link).



Table ContactInfo
- ContactID (id pointing to the Contact element).
- Type (text of limited size, referencing the type of contact information, like
		if it is phone, address, email, facebook, g+, etc).
- Data (text of limited size, referencing to the data itself, like the phone number, 
		address, email, link to the facebook account, g+, etc).


Table Contact
- ID (unique from Table element).
- Name (char limited size).
- Desc (char limited size).
- Age (date).
- Type (enum to determine if it is a person / local / whatever)
- Location (text limited size) // this could be simulated with a tag



Table Tag
- ID (primary unique, the same than Table Element).
- Description (text 100 chars or less).
- Name (text, 12 chars? calculate this).


Table TagEntry
- ID (primary unique, auto increment, not from table Element).
- TagID (id pointing to Table Tag referencing the Tag element).
- ContactID (referencing the ID of the Contact in the table Element).
- Description (limited size text)

Table Link
- ID (primary unique, auto increment, generated by the table Link).
- FirstElement (ID pointing to the first linked element from the table Element).
- SecondElement (ID pointing to the second linked element from the table Element).
- Description (text with limited size)
